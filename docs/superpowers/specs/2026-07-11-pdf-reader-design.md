# PDF 阅读器设计文档

日期：2026-07-11

## 背景与目标

使用 `github.com/lxn/walk`（Windows 原生 GUI 控件库）开发一个 Windows 桌面 PDF 阅读器，定位为"标准阅读器"：支持打开/浏览/缩放/搜索/书签导航/缩略图/多标签页/最近文件/加密文档/命令行打开，暂不包含打印、文本选择复制、批注等高级功能。

## 环境约束

- 本机 Go 工具链位于 `D:\soft\go\bin\go.exe`，为 32 位（`windows/386`）宿主，但已验证可通过设置 `GOARCH=amd64 GOOS=windows` 交叉编译出 64 位 Windows 可执行文件。项目统一按 64 位目标构建，以获得更好的 wazero（WASM 运行时）性能。
- `CGO_ENABLED=0`：整个方案（walk + go-pdfium WASM 模式）均不依赖 cgo，无需安装 C 编译器。

## 技术选型

| 用途 | 选型 | 理由 |
|---|---|---|
| GUI | `github.com/lxn/walk` + `walk/declarative` | Windows 原生控件，纯 Go/syscall 实现 |
| PDF 解析渲染 | `github.com/klippa-app/go-pdfium`（WASM/wazero 模式） | 无需 cgo/C 编译器，基于 PDFium（Chrome 同款引擎），渲染质量高 |
| 资源嵌入 | `go-winres` | 生成 `.syso` 嵌入图标 + manifest（声明 per-monitor DPI 感知 + comctl32 v6，walk 主题控件正常显示的必要条件） |
| 配置持久化 | 本地 JSON 文件 | `%APPDATA%\PDFReader\config.json`，跨用户迁移、调试方便，不依赖注册表 |

## 项目结构

```
PDFReader/
  go.mod
  main.go                 // 入口：解析 os.Args 中的文件路径，初始化 pdfium 池，启动主窗口
  internal/
    config/                // 最近文件列表、窗口状态持久化
    pdfengine/             // go-pdfium 封装：打开/关闭文档、渲染页面为图像、读取大纲、文本搜索
    document/              // 单个已打开文档的运行时状态：当前页、缩放模式、页面渲染缓存、搜索状态
    ui/                    // walk 声明式 UI：主窗口、标签页管理、侧边栏、工具栏、状态栏、各类对话框
  testdata/                // 测试用的小型 PDF 样本（含一份加密样本）
  build/                   // winres 配置、图标资源
```

模块职责边界：
- `pdfengine` 只做"PDF → 数据/图像"的转换，不感知 UI；对外暴露接口，便于用假实现做单元测试。
- `document` 是 `pdfengine` 和 `ui` 之间的中间层，持有每个标签页的状态，不直接操作 walk 控件。
- `ui` 只负责窗口/控件搭建和事件转发，业务逻辑委托给 `document`/`config`/`pdfengine`。

## PDF 渲染流水线

- **单页模式**：一次只显示一页，翻页按钮/快捷键切换，不做整本连续滚动（后续可迭代为连续滚动模式）。
- **渲染方式**：首版本采用同步渲染（在 UI 消息循环中直接调用 `pdfengine` 渲染当前页为 `image.RGBA`，转 `walk.Bitmap` 后交给自定义绘制的页面视图控件），配合容量约 5 页的 LRU 缓存（key 为 `(pageIndex, zoomLevel)`），命中缓存的翻页/返回操作无需重新渲染。
- **预取**：渲染完当前页后，异步在后台 goroutine 预取前一页/后一页写入缓存，通过线程安全方式回传主线程（具体走 walk 的线程封送机制，实现阶段确定 API），不阻塞 UI。
- **缩放**：支持自由百分比（25%–400%，±按钮每次 10%）、"适合宽度"、"适合页面" 三种模式，缩放变化按新 DPI 重新渲染当前页。
- **目录大纲**：打开文档时一次性读取整个 outline 树，构建为 `walk.TreeModel`，点击节点跳转到目标页。
- **缩略图**：懒渲染，仅渲染当前可视区域附近（如 ±10 页）的缩略图，滚动时增量渲染并独立缓存，避免整本书一次性渲染阻塞界面。
- **文本搜索**：Ctrl+F 触发时通过 pdfium 文本查找 API 按需搜索全文档（不预提取全文），命中后跳转到对应页并在页面视图上叠加半透明高亮矩形标出匹配位置，支持 F3/Shift+F3 循环上下一个匹配。
- **加密文档**：打开时若 pdfium 返回"需要密码"错误，弹出密码输入对话框重试；密码错误提示重新输入或取消。

## UI 设计

### 主窗口布局

```
┌─────────────────────────────────────────────────┐
│ 菜单栏: 文件 视图 转到 帮助                          │
├─────────────────────────────────────────────────┤
│ 工具栏: [打开] [◀][▶] [页码/总页数] [－][100%][+] │
│         [适合宽度][适合页面] [🔍查找] [侧边栏开关]   │
├─────────────────────────────────────────────────┤
│ 标签页: [文档A.pdf ×] [文档B.pdf ×] [+]            │
├──────────┬──────────────────────────────────────┤
│ 侧边栏    │                                      │
│ [目录|缩略图]│         页面渲染区域(可滚动)          │
│ 树/列表   │                                      │
│          │                                      │
├──────────┴──────────────────────────────────────┤
│ 状态栏: 第 3 / 120 页    缩放 125%                 │
└─────────────────────────────────────────────────┘
```

### 标签页管理

- 用 `walk.TabWidget` 承载多文档；原生 Windows Tab 控件不带关闭按钮，通过右键菜单「关闭标签」+ 快捷键 Ctrl+W + 鼠标中键点击标签三种方式关闭。
- 关闭最后一个标签后侧边栏/工具栏切换为禁用态，主窗口不自动退出。
- 每个标签页持有独立的 `document.Document` 状态（当前页、缩放、侧边栏选中模式、搜索状态互不干扰）。

### 侧边栏

- 一个 `Composite` 内嵌小型 `TabWidget`：「目录」「缩略图」两页；工具栏 + 视图菜单提供整体折叠/展开开关。

### 查找栏

- 内嵌在页面视图上方（非模态对话框），Ctrl+F 呼出/聚焦；含搜索框、上一个/下一个按钮、"第 X/Y 处匹配"提示；Esc 关闭。

### 对话框

- 打开文件：标准 `walk.FileDialog`（过滤器 `*.pdf`）
- 密码输入：自定义 `walk.Dialog`，`PasswordEdit` 密码框
- 转到页面：Ctrl+G 弹出页码输入框，带范围校验
- 关于：简单信息对话框

### 菜单

- 文件：打开、最近打开的文件（子菜单，最多 10 项）、关闭标签、退出
- 视图：放大、缩小、适合宽度、适合页面、显示/隐藏侧边栏
- 转到：上一页、下一页、首页、末页、跳转到页码
- 帮助：关于

### 快捷键

| 功能 | 快捷键 |
|---|---|
| 打开文件 | Ctrl+O |
| 关闭标签 | Ctrl+W |
| 上一页/下一页 | ←/→ 或 PageUp/PageDown |
| 首页/末页 | Home/End |
| 跳转页码 | Ctrl+G |
| 查找 | Ctrl+F，F3/Shift+F3 上/下一个 |
| 放大/缩小 | Ctrl+=/Ctrl+- |
| 重置缩放（适合页面） | Ctrl+0 |

### 命令行/文件关联

- `main.go` 检查 `os.Args[1]`，若存在且为可读 PDF 文件，启动后直接在新标签页打开。用户可自行将该 exe 设为 `.pdf` 关联程序；应用本身不写注册表。

## 配置持久化

- 路径：`%APPDATA%\PDFReader\config.json`
- 内容：最近打开文件列表（路径 + 最后打开时间，最多 10 条，去重，最新排最前）、上次窗口大小/位置、上次侧边栏展开状态与选中页签。
- 启动读取失败（文件不存在/损坏）→ 静默使用默认配置，不打断启动。
- 保存失败 → 仅记录日志，不弹窗打扰用户。

## 错误处理

- 文件无法打开（不存在/损坏/非 PDF）→ `walk.MsgBox` 错误提示，不影响其他已打开标签页。
- 密码错误 → 密码对话框内联提示"密码错误"，允许重试或取消。
- 最近文件列表中路径失效（文件被删除/移动）→ 点击时提示"文件不存在，是否从列表中移除"，确认后移除该项。
- pdfium/wazero 初始化失败（极端情况）→ 启动时捕获并提示明确错误信息后退出，避免静默崩溃。

## 测试策略

自动化单元测试：
- `config`：JSON 读写往返、最近文件去重与容量裁剪逻辑。
- `document`：缩放计算（适合宽度/适合页面换算为具体缩放比例）、页面缓存 LRU 淘汰逻辑。
- `pdfengine`：基于 `testdata/` 样本 PDF（含一份加密样本）测试打开、页数读取、大纲解析、文本搜索；对外用接口封装，UI 层测试可用假实现替代真实 pdfium 调用。

不做自动化 UI 测试：walk 没有可用于无头 CI 的控件模拟点击机制。实现完成后需手动走查以下场景：打开/翻页/缩放/搜索/书签/缩略图/多标签/加密文件/最近文件/命令行参数打开。

## 范围之外（本期不做）

- 打印
- 文本选择/复制
- 批注/高亮标记（除搜索命中的高亮显示外）
- 连续滚动浏览模式（先做单页模式，作为后续迭代方向）
- 自动修改注册表进行文件关联
